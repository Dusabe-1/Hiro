C - Constants

In C, a constant is a value that cannot be changed once it is assigned. Constants make programs safer, easier to maintain, and more readable.

1. Why Use Constants?

  • Prevent accidental modification of important values.
  • Make code easier to understand.
  • Allow easy updates â€” change the value in one place instead of many.

2. Types of Constants

Type Example Description
Integer Constant 100 Whole numbers without decimal points.
Floating-point Constant 3.14 Numbers with decimal points or in scientific notation.
Character Constant 'A' Single characters enclosed in single quotes.
String Constant "Hello" Sequence of characters enclosed in double quotes.

3. Defining Constants

Using const Keyword

#include <stdio.h>

int main() {
    const float PI = 3.14; // Constant
    int radius = 5;
    float area = PI * radius * radius;

    printf("Area = %.2f\n", area);
    // PI = 3.1415; // Error: Cannot modify constant
    return 0;
}

Output:

Area = 78.50

Using #define Preprocessor Directive

#include <stdio.h>
#define PI 3.14

int main() {
    int radius = 5;
    float area = PI * radius * radius;

    printf("Area = %.2f\n", area);
    return 0;
}

Output:

Area = 78.50

4. const vs #define

Feature const #define
Type Safety Yes, type-checked by compiler No type checking
Scope Follows normal variable scope rules Global (unless undefined later)
Storage Stored in memory Replaced by preprocessor before compilation

5. Constant Expressions

#include <stdio.h>

#define LENGTH 10
#define WIDTH 5
#define AREA (LENGTH * WIDTH)

int main() {
    printf("Area = %d\n", AREA);
    return 0;
}

Output:

Area = 50

6. Enumerated Constants

#include <stdio.h>

enum Week { MON=1, TUE, WED, THU, FRI, SAT, SUN };

int main() {
    enum Week today = WED;
    printf("Day number: %d\n", today);
    return 0;
}

Output:

Day number: 3

Enums create a set of named integer constants.

7. Best Practices

  • Use const for variables with a specific type.
  • Use #define for compile-time values and macros.
  • Always use uppercase names for constants (MAX_VALUE).
  • Keep constants descriptive to improve readability.
